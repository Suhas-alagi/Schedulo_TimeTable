Two Independent Generators

1. PRACTICAL TIMETABLE GENERATOR (Lab Sessions)

Goal: Schedule batch practicals into labs

Key Data Structures:

  • batch_occupied[Year, Division, Batch][Day][Slot] → True/False (marks slot as taken)
  • lab_schedule[Lab][Day][Slot] → List of sessions

Algorithm: 30-pass greedy scheduler

  • Each pass: Iterate through all days → all time slots (11:15, 14:15, 16:20)
  • For each slot, try to place practicals using round-robin fairness (SY gets 2 turns, TY gets 2 turns, BE gets 1 turn — repeating)
  • Stop when no more placements made (stable state)

2. LECTURE TIMETABLE GENERATOR (Class Lectures)

Goal: Schedule individual lectures into class timetables

Key Data Structures:

  • class_timetables[Year, Division] → schedule with Day → Slot → Lectures list
  • Each subject gets N lecture slots equal to its theory hours

Algorithm: Same 30-pass approach

  • Each pass: Days → Slots (10:15 to 16:20, skip lunch at 13:15)
  • Try to place first unscheduled lecture per subject using round-robin ordering
  • Stop when stable

🔒 Constraint Checking

Constraint For Practical For Lecture How it Works
Faculty Busy ✓ ✓ Faculty already teaching in another lab/class at this time
Preferred Off ✓ — Faculty marked this day/slot unavailable
Batch/Class Slot Occupied ✓ ✓ Batch or class already has activity at this slot
Lab Availability ✓ — No free lab OR required lab is occupied
2-Hour Continuation ✓ — For 2-hr practicals: next slot must also be free
Same Subject per Day — ✓ Subject can only have 1 lecture per day (spread constraint)
Faculty Break — ✓ Faculty needs break between back-to-back sessions in same class

⚙️ Round-Robin Fairness

Why it matters: Prevents starvation (e.g., BE year getting locked out)

Cycle Pattern: ['SY', 'SY', 'TY', 'TY', 'BE'] repeating

  • SY gets turn #1 and #2
  • TY gets turn #3 and #4
  • BE gets turn #5
  • Then cycle repeats
Key Benefit: This guarantees BE is visited once every 5 tries, so slots for lower years can't monopolize availability. Every year gets a fair chance at placement.

🎯 Key Algorithm Insights

1. Greedy Strategy — Always take the first valid slot found (no backtracking). This is fast but may leave gaps.
2. Multi-pass Stabilization — Keeps trying up to 30 passes until nothing more fits. Later passes may find gaps created by earlier placements.
3. Fairness First — Round-robin ordering prevents unfair lockouts where one year monopolizes slots.
4. Constraint-Driven — No placement without passing ALL checks. Constraints are binding filters.
5. No Conflict Resolution — Unscheduled items become "leftovers" reported to UI for manual intervention.
